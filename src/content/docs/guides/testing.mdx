Since vimee is built on pure functions, testing is straightforward — no mocks, no DOM, just input and output.

## Using @vimee/testkit

```bash
npm install -D @vimee/testkit
```

```ts
import { testEditor } from "@vimee/testkit";

testEditor("x deletes character under cursor", {
  content: "hello",
  cursor: { line: 0, col: 0 },
  keys: "x",
  expected: {
    content: "ello",
    cursor: { line: 0, col: 0 },
  },
});
```

## Testing Without testkit

You can also test with plain assertions using the core API:

```ts
import { createEditor, processKey } from "@vimee/core";
import { expect, test } from "vitest";

test("enter insert mode with i", () => {
  const editor = createEditor({ content: "hello" });
  const result = processKey(editor, { key: "i" });
  expect(result.mode).toBe("insert");
});
```
